## 1 使用http协议访问web

我们在浏览器输入一个URL，就能访问web页面。 web页面不会凭空显示出来， 根据特定的URL，web浏览器去服务器获取文件资源等信息，从而显示出web页面。

**像这种通过发送请求获取服务器资源的web浏览器等， 都可成为client客户端**

web使用一种规则的约定， 名为HTTP(hyper text transfer protocol， 超文本传输协议)作为规范， 完成一系列流程。

## 2 http的诞生

1989年3月， CERN(欧洲核子研究组织)的蒂姆·伯纳斯-李博士提出了一种能让全世界研究者们共享知识的设想。

1990年11月， CERN成功研发第一台web服务器和web浏览器， 1992年9月， 日本第一个网站主页上线。

1994年12月，网景通信公司发布了netscape navigator浏览器， 1995年8月， 微软发布了IE浏览器。

紧随其后的现在已然成为web标准之一的Apache。

1995左右网景与微软之间的浏览器大战， 一直到2000年的网景公司衰落而告一段落， 可随着2004年的Mozilla基金发布的firefox浏览器， 第二次浏览器大战随之爆发。

另外随着chrome、opera、safari等浏览器的发布， 浏览器大战仍在继续。

**当年，HTTP协议的出现主要是为了解决文本传输的难题， 由于协议本身的简单， 于是在此基础上设想了很多应用方法及扩展。**

## 3 网络基础TCP/IP

了解HTTP，势必要了解TCP/IP协议族，因为HTTP属于它内部的子集。

### 3.1 TCP/IP协议族

计算机与网络设备通信，双方就必须基于 相同的方法比如： 如何探测到通信目标、 哪一边先通信、 哪种语言通信、怎么结束通信等规则都需要事先确定。 不同的硬件、操作系统之间通信，所有的一切都需要一种规则， 这种规则统称为**协议**

:::info

有种说法是TCP/IP是指特定的协议， 还有一种说法则是TCP/IP是互联网相关联的协议的集合， 还有一种则是TCP/IP是IP协议的通信过程中的，使用到的协议族的统称。

:::

### 3.2 TCP/IP分层管理

TCP/IP协议族最重要的一点是分层。 按照层次分别分为以下四层： 应用层、传输层、网络层和网络链路层。

层次化后有以下好处：

1. 如果某些地方需要改变设计，不需要全部替换， 仅改变自己所在的层次即可。

1. 处于应用层的应用只考虑自己的任务，而无需弄清楚对方的运输线路、是否可以传输送达。

TCP/IP的各层作用如下：

- **应用层**：提供应用服务通信时的活动
  - HTTP、FTP、DNS(domain name system 域名系统)等
- **传输层**：对上层应用层提供处于网络连接的两台计算机之间的数据传输
  - TCP(传输控制协议)、UDP(用户数据报协议)等
- **网络(互连)层**：处理网络上流动的数据包(网络传输的最小单位)
    - 规定了通过怎么样的路径(传输路线)到达对方计算机
- **网络链路层**：处理连接网络的硬件部分
    - 操作系统、硬件设备光驱
    - NIC(network interface card 网络接口卡)、网线、交换机等

### 3.3 TCP/IP通信传输流

![tcp/ip传输流](/assets/diagrammatizeHTTP/one1.png)

经过每一层都会被打赏每一层该属的首部信息， 反之在接收端经过层与层传输数据时， 都会把对应的首部去掉。

这种数据信息包装，称之为封装。

## 4 HTTP关系亲密协议： TCP、IP、DNS

### 4.1 传输协议： IP

IP(internet protocol)处于网络层， 基于所有使用网络的系统都用到了IP协议。

:::tip

IP协议与IP地址可不是一个概念， IP地址是IP协议的一个重要组成部分。

:::

IP协议作用是将 数据包传送给对方， 确保可以传送到对方那里，则是需要满足两个条件： IP地址和MAC地址。

IP地址是节点被分配的地址， MAC地址是网卡所属的固定地址， IP和MAC可以进行配对。


#### 使用ARP协议凭借MAC地址通信

通信双方在局域网下情况很少，通过经过多台计算机和网络设备中转才能连接到对方， 在中转时， 利用下一站设备的MAC地址来搜索下一个中转目标， 这时会采用ARP(address resolution protocol 地址解析协议)协议根据IP地址反查MAC地址。

#### 路由选择

在到达通信目标的中转过程中，计算机和路由器等网络设备都是获悉粗略的传输路线， 这种机制叫做**路由选择**。

### 4.2 确保可靠传输协议： TCP

TCP(传输控制协议)处于传输层， 提供可靠的字节流服务。

:::tip

字节流服务就是为了方便运输，将大块数据分割成以报文段为单位的数据包进行管理。

:::

TCP协议**分割数据**， 且**确认数据最终是否送达到对方**

#### 确保数据能达到目标

TCP协议为了可以准确数据达到目标， 采用了**三次握手**， 不会对传送后的情况置之不理。

![三次握手](/assets/diagrammatizeHTTP/one2.png)

## 5 应用层：DNS服务

DNS(domain name system 域名系统)处于应用层， 负责将域名转换为IP地址。

相对于ip地址的一串数字， 主机名和域名的可读性更好， 但计算机容易处理一长串数字， DNS可以通过域名查IP，或者反查。
